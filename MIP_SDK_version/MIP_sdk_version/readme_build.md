# Building the Official MicroStrain MIP SDK for Teensy 4.1

This note explains how this project compiles the official MicroStrain MIP SDK for a Teensy 4.1 using PlatformIO and the Arduino framework.

The working PlatformIO project is:

```text
MIP_SDK_version/MIP_sdk_version
```

The application entry point is:

```text
src/main.cpp
```

The embedded copy of the MIP SDK used by PlatformIO is:

```text
lib/mip_sdk
```

In the original discussion, `mip_sdk_full` was the full official SDK clone. The PlatformIO project uses a smaller vendored SDK copy under `lib/mip_sdk`, with only the source needed for the Teensy build.

## Build Environment

The project is configured for Teensy 4.1:

```ini
[env:teensy41]
platform = teensy
board = teensy41
framework = arduino

monitor_speed = 115200
monitor_eol = LF
```

PlatformIO reported this hardware/toolchain during the successful build:

```text
PLATFORM: Teensy > Teensy 4.1
HARDWARE: IMXRT1062 600MHz, 512KB RAM, 7.75MB Flash
framework-arduinoteensy
tool-teensy
toolchain-gccarmnoneeabi-teensy
```

## PlatformIO SDK Configuration

The project includes the SDK as a local PlatformIO library:

```ini
lib_deps = file://lib/mip_sdk
```

The project-level build flags are:

```ini
build_flags =
    -std=gnu++14
    -Wall
    -O2
    -Ilib/mip_sdk/src
    -DMICROSTRAIN_PLATFORM_EMBEDDED=1
    -DMIP_USE_EXTRAS=0
```

The SDK library manifest is:

```text
lib/mip_sdk/library.json
```

It limits the build to the C core of the SDK:

```json
"srcFilter": [
  "+<microstrain/**/*.c>",
  "+<mip/**/*.c>",
  "-<**/connections/**>",
  "-<**/extras/**>",
  "-<**/*_test*>",
  "-<**/*_Test*>"
]
```

This is important for embedded builds because the desktop connection layers, extras, and tests are not needed on Teensy and may pull in host-specific dependencies.

## Main Problems Found During Build

The first compile attempt failed with:

```text
fatal error: microstrain/embedded_time.hpp: No such file or directory
```

The root causes were:

1. The `lib/mip_sdk` copy was trimmed from the official SDK and did not include `microstrain/embedded_time.hpp`, which is needed when SDK headers are included from Arduino C++ code.
2. `mip_version.h` is normally generated by the SDK CMake build. PlatformIO compiles the source directly, so this generated header must be provided manually.
3. In official MIP SDK v4, the C API is namespaced as `mip::C` and `microstrain::C` when included from C++.
4. Some callback signatures, function signatures, and field names in `main.cpp` needed to match SDK v4.

## Required Fixes

### 1. Add `embedded_time.hpp`

File:

```text
lib/mip_sdk/src/microstrain/embedded_time.hpp
```

Current compatibility shim:

```cpp
#pragma once

#include <microstrain/embedded_time.h>

namespace microstrain
{

using EmbeddedTimestamp = C::microstrain_embedded_timestamp;
using EmbeddedTimeout   = C::microstrain_embedded_timeout;

} // namespace microstrain
```

This lets the SDK headers compile cleanly when included by Arduino C++ code.

### 2. Add `mip_version.h`

File:

```text
lib/mip_sdk/src/mip/mip_version.h
```

Current generated version header:

```cpp
#pragma once

// Generated from the official SDK version tag for the PlatformIO build.
#define MIP_SDK_VERSION_FULL "v4.0.0"
#define MIP_SDK_VERSION "4.0.0"
#define MIP_SDK_VERSION_MAJOR 4
#define MIP_SDK_VERSION_MINOR 0
#define MIP_SDK_VERSION_PATCH 0
```

If you update the official SDK clone to a different tag, update this file to match that tag.

### 3. Use the SDK v4 C Namespaces in `main.cpp`

Because the Arduino application is C++, the SDK C API is used through these namespaces:

```cpp
using namespace mip::C;
using namespace microstrain::C;
```

This allows code such as `mip_interface`, `mip_time`, `mip_cmd_result`, and the generated command helpers to resolve correctly.

### 4. Use SDK v4 Callback Signatures

The MIP interface in `main.cpp` is initialized with callbacks that bridge SDK I/O to `Serial2`, the UART connected to the CV7/GV7 IMU:

```cpp
mip_interface_init(&mip_dev,
                   100,
                   1000,
                   &app_mip_send_callback,
                   &app_mip_recv_callback,
                   &app_mip_update_callback,
                   NULL);
```

The send callback writes bytes to the IMU:

```cpp
Serial2.write(data, length);
Serial2.flush();
```

The receive callback reads pending bytes from the IMU and timestamps them using `millis()`.

### 5. Use SDK v4 Command Helpers

The code uses SDK helpers for one-time CV7/GV7 setup:

```cpp
mip_aiding_write_frame_config(...)
mip_filter_write_aiding_measurement_enable(...)
mip_3dm_save_device_settings(...)
```

It also uses SDK packet and serializer helpers for realtime external-aiding messages:

```cpp
mip_packet_create(...)
mip_packet_add_field(...)
mip_packet_finalize(...)
insert_mip_aiding_pos_llh_command(...)
insert_mip_aiding_vel_ned_command(...)
insert_mip_aiding_heading_true_command(...)
```

This keeps the packet header, field header, payload serialization, and checksum consistent with the official MIP SDK.

## Build Command

From the project directory:

```powershell
cd MIP_SDK_version\MIP_sdk_version
pio run -e teensy41
```

If PlatformIO is installed only inside a user virtual environment, use the full path shown by your installation. In the original build log, the verified command was:

```powershell
C:\Users\rqy19\.platformio\penv\Scripts\pio.exe run -e teensy41
```

Expected successful output:

```text
SUCCESS
```

The firmware image is generated at:

```text
.pio/build/teensy41/firmware.hex
```

## Upload Command

With the Teensy 4.1 connected by USB:

```powershell
pio run -e teensy41 -t upload
```

## Monitor Command

Use the serial monitor only when `MIRROR_MIP_TO_USB` is set to `false` in `src/main.cpp`, because the default configuration mirrors binary MIP packets to USB:

```powershell
pio device monitor -b 115200
```

If `MIRROR_MIP_TO_USB` is `true`, capture the USB stream as binary data instead of opening a text monitor.

## Troubleshooting

| Error / symptom | Cause | Fix |
| --- | --- | --- |
| `microstrain/embedded_time.hpp: No such file or directory` | Trimmed SDK copy is missing a C++ compatibility header | Add `lib/mip_sdk/src/microstrain/embedded_time.hpp` |
| `mip_version.h` missing | PlatformIO does not run the SDK CMake generation step | Add `lib/mip_sdk/src/mip/mip_version.h` manually |
| SDK symbols not found from `main.cpp` | SDK v4 C API is inside C++ namespaces | Add `using namespace mip::C;` and `using namespace microstrain::C;` |
| Callback type mismatch | `mip_interface_init()` expects SDK v4 callback signatures | Match `app_mip_send_callback`, `app_mip_recv_callback`, and `app_mip_update_callback` to the SDK headers |
| Host connection files fail to compile | Full SDK includes desktop serial/TCP layers | Use `library.json` `srcFilter` to exclude `connections`, `extras`, and tests |
| Text monitor shows unreadable bytes | USB is mirroring binary MIP packets | Set `MIRROR_MIP_TO_USB = false` for status text, or capture binary output |

## Summary

The official MIP SDK can be built for Teensy 4.1 with PlatformIO by vendoring the SDK as a local library, compiling only the embedded C core, adding the two headers normally expected from the full/CMake SDK flow, and adapting `main.cpp` to the SDK v4 C API namespaces and function signatures.

The verified build output was:

```text
.pio/build/teensy41/firmware.hex
```
